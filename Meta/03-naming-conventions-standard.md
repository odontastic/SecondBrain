# Resource Naming Conventions Standard

## Overview
This document outlines the standardized naming conventions for Resources in the LifeOS system.

## Principles
- Use descriptive names that clearly indicate the content of the resource.
- Use lowercase letters with words separated by hyphens (-) or underscores (_).
- Avoid special characters and spaces.

## Examples
- `effective_strategies_self_improvement.md`
- `life-os-blueprint-v1.md`
- `personal-hair-color-guide.md`

## Guidelines
- Be consistent in using either hyphens or underscores as word separators.
- Use meaningful and descriptive names that provide context.
- Keep file names concise but informative.

## Maintenance
This standard will be reviewed and updated quarterly to ensure it remains relevant and effective.
